Paste your game's result history and press Begin. The engine learns it and posts its 8 most-likely numbers for the next round — check them against the live app. Tap each result that comes up; it's saved and the engine relearns, sharpening over a 7–8 day test. The same engine aimed at Sylox's commit-reveal result gets nothing: the seed is sealed before the bet, so there is no state to learn and its tightest 4-number guess just evades.

What you're looking at A genuine learning predictor, running entirely in your browser. It surfaces real structure when it exists — a weak or recoverable RNG is exposed within a few hundred rounds — and honestly sits at the no-skill baseline when a stream is fair, with a 95% interval so a fair game can't read as a false positive. How it works. The predictor is a self-tuning ensemble (recency-weighted frequency, order-1/2 Markov, gap and periodicity signals) whose weights track each signal's own recent accuracy. It sees only results — never engine internals — so its hit rate on the live game is yours to verify. Skill is measured against a fixed central-8 cover (the no-skill baseline) with a 95% interval, so a fair stream reads as zero edge, not a false positive. The right engine is Sylox's commit-reveal CSPRNG: result = HMAC-SHA256(serverSeed, clientSeed:nonce); commit = SHA-256(serverSeed) is published before the bet and the seed revealed after for verification — no recoverable state, so the same engine finds no edge. Nothing is done to the player. All computation runs offline in your browser.
